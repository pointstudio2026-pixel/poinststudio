# Golden Rules

-   문서가 우선이다.
-   추측하지 않는다.
-   임의 기능을 추가하지 않는다.
-   기존 구조를 존중한다.
-   테스트 없이 완료 처리하지 않는다.

------------------------------------------------------------------------

# Architecture Rules

-   Clean Architecture
-   Domain Driven Design
-   SOLID
-   TypeScript Strict Mode
-   Adapter Pattern
-   Repository Pattern
-   Use Case Pattern

Route Handler에는 비즈니스 로직을 작성하지 않는다.

------------------------------------------------------------------------

# Coding Standards

-   any 사용 금지(불가피한 경우 제외)
-   명확한 타입 정의
-   함수는 단일 책임 원칙 준수
-   공통 로직은 재사용 가능한 모듈로 분리

# Documentation Rules

문서를 무시하고 구현하지 않는다.

# Definition of Complete

다음 조건을 모두 만족해야 완료로 간주한다.

-   기능 구현
-   테스트 통과
-   타입 오류 없음
-   Lint 통과
-   문서 일치
-   보안 규칙 준수

------------------------------------------------------------------------

# Forbidden

-   문서와 다른 기능 추가
-   하드코딩된 비밀키
-   Provider 종속 구조
-   중복 코드
-   테스트 없는 완료 선언
