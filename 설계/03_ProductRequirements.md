# Purpose

이 문서는 ASTER가 반드시 제공해야 하는 핵심 제품 요구사항을 정의한다.
세부 구현은 이후 PRD를 따르며, 본 문서는 기능 범위와 성공 기준을
결정하는 기준 문서이다.

------------------------------------------------------------------------

# Product Objective

ASTER는 디자이너가 브랜드 방향성을 빠르게 탐색하고, 참고용 컨셉을
생성하여 최종 디자인 작업 시간을 단축하도록 돕는다.

------------------------------------------------------------------------

# Target Users

## Primary

-   프리랜서 디자이너
-   브랜드 디자이너
-   디자인 에이전시

## Secondary

-   디자인 전공 학생
-   스타트업 인하우스 디자이너

------------------------------------------------------------------------

# Functional Requirements

## FR-001 Project Workspace

-   프로젝트 생성
-   프로젝트 저장
-   버전 관리
-   즐겨찾기

## FR-002 Brand Interview

-   AI 질문 생성
-   사용자 답변 수집
-   Brand Brief 생성

## FR-003 Brand Strategy

-   브랜드 가치
-   톤앤매너
-   포지셔닝
-   핵심 키워드

## FR-004 Style Engine

-   150개 이상 스타일
-   대분류 / 중분류 / 소분류
-   스타일 추천

## FR-005 Image Generation

-   참고용 컨셉 이미지 생성
-   구독 플랜별 생성 수 제한

## FR-006 One Click Edit

-   더 심플하게
-   더 럭셔리하게
-   컬러 변경
-   심볼 변경
-   타이포 변경

## FR-007 Concept Board

-   컬러 팔레트
-   스타일 키워드
-   브랜드 설명
-   추천 방향

## FR-008 Mockup

-   명함
-   간판
-   패키지
-   모바일 화면

------------------------------------------------------------------------

# Non Functional Requirements

-   빠른 응답성
-   확장 가능한 구조
-   AI 모델 교체 가능
-   안정적인 프로젝트 저장
-   법적 리스크 최소화

------------------------------------------------------------------------

# Success Criteria

-   첫 브랜드 방향 제안 3분 이내
-   첫 컨셉 생성 15분 이내
-   높은 무료→유료 전환율
-   작업 시간 절감

------------------------------------------------------------------------

# Out of Scope (MVP)

-   팀 협업
-   실시간 공동 편집
-   모바일 앱
-   브랜드 적합성 점수
-   플러그인 생태계
