# 19_PRD_Subscription

Project: ASTER Version: 3.1 (AI-Native Specification) Status: Draft

------------------------------------------------------------------------

# Feature ID

PRD-014

Feature Name

Subscription & Billing

Priority

P0 (Business Core)

------------------------------------------------------------------------

# Goal

구독 시스템은 사용자에게 합리적인 사용량을 제공하면서 서비스의 목표
원가율(20% 이하)을 유지하도록 설계한다.

------------------------------------------------------------------------

# Product Principles

-   기능보다 워크플로의 가치를 판매한다.
-   사용량은 투명하게 표시한다.
-   플랜 업그레이드는 자연스럽게 유도한다.

------------------------------------------------------------------------

# Plans

## Free

-   프로젝트 생성 제한
-   일일 생성 제한
-   기본 해상도
-   기본 스타일 사용

## Pro

-   생성량 증가
-   고해상도
-   빠른 생성 큐
-   모든 스타일 사용

## Studio

-   최고 생성량
-   팀 기능(향후)
-   우선 처리
-   신기능 우선 제공

------------------------------------------------------------------------

# Usage Policy

측정 대상

-   이미지 생성
-   원클릭 수정
-   목업 생성
-   내보내기

사용량은 월 단위 초기화한다.

------------------------------------------------------------------------

# Cost Control

-   목표 원가율: 20% 이하
-   Provider별 비용 모니터링
-   캐시 우선 사용
-   동일 요청 중복 생성 방지
-   저비용 모델 우선 사용 후 필요 시 고성능 모델 사용

------------------------------------------------------------------------

# Upgrade Triggers

-   생성 한도 초과
-   고급 스타일 선택
-   고해상도 요청
-   빠른 큐 사용

------------------------------------------------------------------------

# Billing Flow

회원가입 → Free 시작 → 업그레이드 → 결제 → 플랜 활성화 → 사용량 추적 →
갱신

------------------------------------------------------------------------

# API

GET /subscription

POST /subscription/upgrade

GET /subscription/usage

POST /billing/webhook

------------------------------------------------------------------------

# Database

subscriptions plans usage_logs billing_events

------------------------------------------------------------------------

# Error Handling

SUB-001 Payment Failed

SUB-002 Usage Limit Reached

SUB-003 Invalid Plan

------------------------------------------------------------------------

# Acceptance Criteria

-   플랜 변경 가능
-   사용량 정확히 표시
-   한도 초과 시 안내
-   업그레이드 정상 처리

------------------------------------------------------------------------

# Definition of Done

-   Billing API 연동
-   Usage Tracking 구현
-   플랜 제한 구현
-   테스트 완료
-   문서 업데이트

------------------------------------------------------------------------

# Claude Code Instructions

구독 제한은 프론트엔드가 아닌 서버에서 검증한다. 사용량은 이벤트
기반으로 기록한다. 원가율 분석을 위한 로그를 남길 수 있도록 설계한다.
